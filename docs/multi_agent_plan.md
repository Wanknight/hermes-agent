## 一、项目概述

### 1.1 项目背景

参考 Edict 项目的"三省六部制"多Agent协作系统，在 Hermes Agent 内实现类似的多Agent协作能力。

### 1.2 核心目标

1. **可切换模式**：支持默认单Agent模式和多Agent模式切换
2. **角色分工**：实现 11 个专用 Agent 角色
3. **流程可控**：支持审议、封驳、重试等完整流程
4. **可观测性**：提供 Dashboard API 查看任务状态
5. **状态持久化**：任务历史可追溯

### 1.3 设计原则

- 最小侵入：复用 Hermes 现有能力（delegate_tool, todo, memory）
- 可插拔：模式切换不影响原有功能
- 零配置启动：默认使用 SQLite + 内存队列
- 渐进增强：可选 Redis/PostgreSQL/Dashboard

```
│  方式3: 环境变量（临时切换）                                  │
│  ─────────────────                                          │
│  HERMES_MULTI_AGENT=true hermes chat                        │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

```yaml

  enabled: false                    # 是否启用多Agent模式
  mode: "default"                   # 模式：default | three_provinces | custom
  orchestrator_model: ""            # 调度器专用模型（空则用主模型）
  
  # Agent 角色池配置
  agents:
    taizi:
      enabled: true
      model: ""                     # 空则继承主模型
    zhongshu:
      enabled: true
      model: ""
    menxia:
      enabled: true
      model: ""
    shangshu:
      enabled: true
      model: ""
    hubu:
      enabled: true
      model: ""
    bingbu:
      enabled: true
      model: ""
    libu:
      enabled: true
      model: ""
    xingbu:
      enabled: true
      model: ""
    gongbu:
      enabled: true
      model: ""
    libu_hr:
      enabled: true
      model: ""
    zaochao:
      enabled: true
      model: ""
  
  # 流程配置
  workflow:
    max_review_rounds: 3            # 门下省最多封驳轮次
    auto_approve_final: true        # 第N轮强制通过
    timeout_per_agent: 300          # 单Agent超时（秒）
  
  # 状态存储
  state:
    backend: "sqlite"               # sqlite | redis | postgres
    redis_url: ""                   # redis://localhost:6379/0
    postgres_url: ""                # postgresql://...
  
  # 事件总线
  event_bus:
    backend: "memory"               # memory | redis
    redis_url: ""
```

### 3.2 角色协作流程

```
用户消息 → 太子(分类) → 中书省(规划) → 门下省(审议) → 尚书省(派发) → 六部(执行)
                ↑____________封驳循环____________↓
```
