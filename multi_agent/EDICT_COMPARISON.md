## 一、项目概况对比

| 维度 | Edict (开源) | 当前实现 (Hermes内置) |
|------|-------------|---------------------|
| **定位** | 独立产品 | Hermes Agent 内置模式 |
| **Agent数量** | 12个 (11业务+1兼容) | 11个 |
| **前端** | React 18 + TypeScript | 无 (CLI/Gateway) |

| **部署方式** | Docker/systemd | Hermes 启动即用 |

## 四、战略决策

### 4.1 定位差异

| 维度 | Edict | 当前实现 |
|------|-------|----------|
| **目标用户** | 非技术用户 | 开发者/技术用户 |
| **使用方式** | Web UI | CLI/IM消息 |
| **集成度** | 独立部署 | 深度集成Hermes |
| **扩展性** | 框架级 | Agent内部模式 |

### 4.2 各自优势

**Edict优势**:
- 完整的可视化Dashboard
- 非技术用户友好
- 一键Docker部署

**当前实现优势**:
- 无需独立部署，开箱即用
- 深度集成Hermes工具链
- Gateway多平台支持(飞书/微信/Telegram)
- 更轻量，零额外依赖
- 支持定期报告（日报/周报）

### 4.3 设计决策

**不需要开发的功能**:

| 功能 | 原因 |
|------|------|
| React Dashboard | Gateway已支持多平台，投入产出比低 |
| 新闻聚合 | 非核心功能 |
| 朝堂议政 | 非核心功能 |

**可选增强的功能**:

| 功能 | 触发条件 |
|------|----------|
| Redis事件总线 | 需要分布式部署时 |
| WebSocket推送 | 需要Web实时更新时 |
| REST API | 需要第三方集成时 |
